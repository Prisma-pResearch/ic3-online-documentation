# Core concepts
A few key concepts when working with IC<sup>3</sup> datasets.

:::note
The key concepts are defined for Epic datasets, for OMOP version data, please refer to [OMOP CDM](https://www.ohdsi.org/data-standardization/).
:::

## Identifiers

| Identifier Name     | Description                                                              | Notes                         |
|---------------------|--------------------------------------------------------------------------|-------------------------------|
| patient_deiden_id   | Deidentified MRN (UF): Patient's Medical Record Number                   | Each dataset has it's own IDs |
| encounter_deiden_id | Deidentified Encounter # (CSN): Patient Encounter Number                 | Each dataset has it's own IDs |
| merged_enc_id       | Merged Encounter IDs based on Encounter Admission and Discharg Timestamp |                               |
| or_case_num         | Operation Case Identical Number                                          |                               |

:::tip
All of tables are built based on subjects(Patients). 
To filter sub-cohort of your project, please use **Encounters**, **or_case_schedule_cnounters** tables firstly to filter the subjects, and then merging with other tables to get full datasets.
:::
